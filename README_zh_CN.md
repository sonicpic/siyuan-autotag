为思源笔记提供可审阅的 AI 分类标签建议。

![AutoTag 预览图](preview.png)

AutoTag 面向已经拥有一套分类标签体系的思源笔记用户。它不会自由生成一套新的标签体系，而是只在你指定的分类标签范围内进行判断，并在写回前保留人工审阅环节。

它尤其适合这样的工作流：先通过 daily notes 或其他无压方式持续记录，再在后续整理阶段将真正进入知识库的内容补上稳定的分类标签。对于不希望纳入分类体系的日记文档，也可以直接排除，不参与分析与扫描。

## 适用场景

- 已经建立了较稳定的分类标签体系
- 笔记数量较多，手动逐篇分类成本较高
- 使用 daily notes 作为记录入口，希望在后续整理阶段补充分类
- 希望借助 AI 提效，但不希望模型直接改写现有标签体系
- 希望集中处理“尚未进入分类系统”的笔记

## 如何工作

AutoTag 会读取当前笔记内容，并通过你配置的大模型接口生成分类建议。它只会在你指定的现有分类标签范围内进行判断，不会自由生成新标签。你也可以为关键标签补充描述，帮助模型理解分类边界。

它既可以分析当前笔记，也可以扫描笔记本中的未分类笔记。所有建议都需要人工审阅后才会写回，而不是由模型直接修改标签。

写回时，AutoTag 只修改文档级 `tags` 属性，并且只处理受管控范围内的分类标签；范围外的既有标签会原样保留。

这让它很适合与 daily notes 配合使用：记录阶段保持轻量，整理阶段再补充分类；不希望进入分类体系的日记文档也可以直接排除。

## 快速开始

1. 打开插件设置，选择一个提供商预设
2. 填写 `API Key`
3. 按需覆盖 `Base URL` 与 `Model`
4. 选择 AutoTag 允许使用的分类标签范围
5. 可选：为关键标签补充描述，说明适用范围和边界
6. 在当前笔记执行 `分析当前笔记`，或通过右键顶栏图标扫描笔记本中的未分类笔记
7. 审阅建议结果后再写回标签

## 使用建议

为了获得更稳定的分类结果，建议优先保证以下几点：

### 1. 分类标签范围尽量清晰

AutoTag 更适合在边界明确的标签集合中工作。如果标签之间语义高度重叠，模型即使能给出建议，结果也容易波动。

### 2. 为关键标签补充描述

对于容易混淆的分类，补充几句简短说明通常能明显改善判断质量。尤其是在标签名称较抽象、或同一上位分类下存在多个近义子类时，这一点很重要。

### 3. 不要强求所有日记参与分类

daily notes 往往承载大量过程性、混合性内容。对这类文档，排除往往比强行分类更合适。更推荐的方式是：先记录，再将值得沉淀的内容整理为独立笔记后参与分类。

### 4. 把它当作审阅辅助，而不是最终裁决者

AutoTag 的价值在于缩小人工判断范围，而不是替代你的分类标准。越清晰的标签体系，越适合发挥它的作用。
